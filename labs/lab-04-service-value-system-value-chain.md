# Lab 04 - Service Value System and Service Value Chain

## Objectives

- Explain the ITIL service value system.
- Identify service value chain activities.
- Map a value stream.
- Connect demand, opportunity, and value.
- Explain governance and continual improvement in the SVS.

## Scenario

BrightDesk receives demand for faster onboarding of new employees. You must map how this demand becomes value through the service value chain.

## Steps

### 1. Identify SVS Components

Write the components:

```text
Guiding principles
Governance
Service value chain
Practices
Continual improvement
```

### 2. List Value Chain Activities

Write:

```text
Plan
Improve
Engage
Design and transition
Obtain/build
Deliver and support
```

### 3. Map the Onboarding Value Stream

Create a flow:

```text
Manager submits new hire request
HR confirms employee details
IT prepares account and laptop
Service desk communicates status
Employee receives access
Manager confirms productivity
Feedback is reviewed
```

Map each step to a value chain activity.

### 4. Identify Inputs and Outputs

For each activity, document:

| Activity | Input | Output |
| --- | --- | --- |
| Engage | Manager request | Confirmed requirements |
| Obtain/build | Approved request | Configured laptop and accounts |

### 5. Add Governance and Improvement

Answer:

1. Who approves the onboarding policy?
2. What metrics show whether onboarding creates value?
3. How is feedback captured?
4. What improvement would you test first?

## Validation

You should have an SVS component list, value chain activity list, onboarding value stream, and governance/improvement notes.

## Checkpoint Questions

1. What is the purpose of the service value system?
2. How is the service value chain different from a fixed process?
3. Where does continual improvement fit?
4. What is the difference between demand and value?

## Exam Focus

Expect questions that ask you to identify value chain activities and explain how SVS components work together.
