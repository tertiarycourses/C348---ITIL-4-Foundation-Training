# Lab 02 - Seven Guiding Principles and Decision Scenarios

## Objectives

- List the seven ITIL guiding principles.
- Apply guiding principles to service decisions.
- Avoid overcomplicated improvement plans.
- Explain why principles are universal and adaptable.

## Scenario

BrightDesk wants to replace its service desk tool because users are unhappy. A manager wants to buy a new platform immediately. You must apply ITIL guiding principles before making a recommendation.

## Steps

### 1. List the Guiding Principles

Write the seven principles:

```text
Focus on value
Start where you are
Progress iteratively with feedback
Collaborate and promote visibility
Think and work holistically
Keep it simple and practical
Optimize and automate
```

### 2. Apply Each Principle

Create a table:

| Principle | Scenario Application |
| --- | --- |
| Focus on value | Identify what users need improved before replacing tools |
| Start where you are | Review current ticket data and user feedback |

Complete all seven rows.

### 3. Diagnose the Service Desk Issue

List evidence you would collect:

- Ticket volume.
- Response time.
- Resolution time.
- Reopened tickets.
- User satisfaction.
- Common request categories.
- Tool usability feedback.


### 5. Decide Whether to Automate

Answer:

1. What should be optimized before automation?
2. What should not be automated yet?
3. What simple automation could help users?

## Validation

You should have a completed principles table, evidence list, pilot improvement, and automation decision.

## Checkpoint Questions

1. Why should organizations start where they are?
2. What does collaborate and promote visibility mean?
3. Why should optimization happen before automation?
4. Which principle prevents unnecessary complexity?

## Exam Focus

Guiding principle questions often describe a situation and ask which principle is being applied.
