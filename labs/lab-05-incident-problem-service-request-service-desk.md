# Lab 05 - Incident, Problem, Service Request, Service Desk

## Objectives

- Distinguish incidents, problems, service requests, and known errors.
- Classify tickets correctly.
- Explain the service desk role.
- Create a simple escalation and communication workflow.

## Scenario

BrightDesk receives the following contacts in one morning:

1. User cannot access email.
2. User asks for a standard software installation.
3. Multiple users report slow VPN.
4. A recurring laptop blue-screen issue has no permanent fix yet.
5. A manager asks for a new hire laptop.

## Steps

### 1. Classify the Contacts

Create a table:

| Contact | Classification | Reason |
| --- | --- | --- |
| User cannot access email | Incident | Unplanned interruption to service |

Classify all five contacts.

### 2. Define Key Terms

Write definitions for:

```text
Incident
Problem
Known error
Workaround
Service request
Service desk
```

### 3. Design an Incident Workflow

Use:

```text
Log
Categorize
Prioritize
Initial diagnosis
Escalate if needed
Resolve
Close
Capture learning
```

### 4. Design a Service Request Workflow

Use:

```text
Submit request
Validate entitlement
Approve if required
Fulfil using standard procedure
Notify user
Close request
```

### 5. Explain Service Desk Value

Write how the service desk:

- Acts as a single point of contact.
- Captures demand.
- Communicates status.
- Supports users.
- Provides data for improvement.

## Validation

You should have a ticket classification table, definitions, incident workflow, request workflow, and service desk value note.

## Checkpoint Questions

1. What is the difference between an incident and a problem?
2. What is a workaround?
3. Why should service requests be standardized?
4. What is the service desk's value?

## Exam Focus

ITIL Foundation often tests precise distinctions between incident, problem, known error, workaround, and service request.
