# Lab 01 - Service Management, Value Co-Creation, Utility, Warranty

## Objectives

- Define service management.
- Distinguish service, product, service offering, output, and outcome.
- Explain value co-creation.
- Distinguish utility and warranty.
- Map cost and risk in a service relationship.

## Scenario

BrightDesk Services provides an Employee Digital Workplace service. Staff use laptops, email, chat, file sharing, HR systems, and finance applications. Users complain that IT talks about tickets and systems, while employees care about staying productive.

## Steps

### 1. Identify the Service

Write:

```text
Service name:
Service provider:
Service consumer:
Users:
Other stakeholders:
```

### 2. Separate Output and Outcome

Create a table:

| Item | Example |
| --- | --- |
| Output | A configured laptop is delivered |
| Outcome | New employee can start productive work on day one |

Add at least three more examples.

### 3. Define Utility and Warranty

For the digital workplace service, answer:

1. What does the service do?
2. Is it fit for purpose?
3. Is it available when needed?
4. Does it have enough capacity?
5. Is it secure enough?

Map the answers to utility and warranty.

### 4. Map Value Co-Creation

List what each party contributes:

| Party | Contribution |
| --- | --- |
| IT provider | Tools, support, knowledge, service desk |
| Employees | Requirements, feedback, responsible usage |
| Managers | Priorities, approvals, escalation context |
| Vendors | Platforms, support agreements, updates |

### 5. Identify Cost and Risk

Document:

- Cost removed from users by the service.
- Cost introduced by using the service.
- Risk removed from users by the service.
- Risk introduced by using the service.

## Validation

You should have a service map, output/outcome table, utility/warranty notes, and value co-creation map.

## Checkpoint Questions

1. What is the difference between output and outcome?
2. Why is value co-created rather than delivered one-way?
3. What is utility?
4. What is warranty?

## Exam Focus

ITIL Foundation questions often test exact definitions. Make sure you can explain utility as fit for purpose and warranty as fit for use.
